1. Подключение по ssh к серверу
	Выполняется из командной строки
	Win + R
	В окне "Выполнить" ввести cmd
	Нажать OK

	- По паролю
		ssh <username>@10.128.1.170
		<Ввод пароля> его не видно при вводе, это нормально

	- По публичному ключу
		Создание файла конфигурации
		1. Открываем каталог .ssh
		2. Создаём файл "config" (без расширения)
		3. Открываем в блокноте (или любом текстовом редакторе)
		4. Пишем следующее
			Host <hostmarker> - любое удобное название 
			  Hostname <hostname> - доменное имя или ip адрес того к чему подключаемся по ssh
			  User <username> - имя пользователя
			  PubKeyAuthentication yes - установка идентификации по публичному ключу
			  IdentityFile <C:\абсолютный\путь\до\файла> - файл с приватным ключом, предоставленный преподаваетлем
		
		Вход
			ssh <hostmarker>


2. Копирование файла на сервер
	- Необходима настроенная по публичному ключу аутентификация
		1. Скопировать файл или папку
		scp -Br С:\Путь\до\файла_или_папки <hostmarker>:/путь/до/папки/назначения
		2. Скопировать содержимое папки
		scp -Br С:\Путь\до\папки\* <hostmarker>:/путь/до/папки/назначения

3. Копирование файлов с сервера
	- Необходима настроенная по публичному ключу аутентификация
		1. Скопировать файл или папку
		scp -Br <hostmarker>:\Путь\до\файла_или_папки C:/путь/до/папки/назначения
		2. Скопировать содержимое папки
		scp -Br <hostmarker>:\Путь\до\папки\* С:/путь/до/папки/назначения

4. Компиляция файла
	mpicc <исходный_файл_с_кодом> -o <результирующий_файл>

5. Создание "rankfile"
	Данный файл позволяет задать количество задействованных процессоров и процессов для каждого узла
	rank <MPI_Rank>=<address> slot=<socket_number>:<logical_cores_number>

	Пример:
	rank 0=0.0.0.0 slot=1:0-2	-	данная строчка указывает что на узле с адресом 0.0.0.0
									будет задействованы логические ядра с
									номерами 0-2 (то же самое что 0,1,2)
									в логическом сокете номер 1
									для обслуживания процесса с MPI_Rank = 0
	rank 1=1.1.1.1 slot=0:0,1
	rank 2=2.2.2.2 slot=1-2		-	здесь же будут задействованы логические ядра с номерами 1,2

	Внутри и вне сокета нумерация ядер различная.
	Т.е. в первой строке задаются ядра с их порядковым номером в сокете,
	а во второй строке порядковый номер логических ядер в системе.


6. Запуск файла
	mpirun -rf <путь_до_rankfile> <путь_до_исполняемого файла> [аргументы]
